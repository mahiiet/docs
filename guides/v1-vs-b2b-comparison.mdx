## Authentication Comparison

### V1 Authentication
- **Method**: SIWE (Sign-In with Ethereum)
- **Endpoints**: 
  - `GET /v1/api/siwe` - Get SIWE message
  - `POST /v1/api/siwe` - Verify signature and get JWT
- **Headers**: `X-Auth-Token: <jwt>`

### B2B Authentication
- **Method**: SIWE (Sign-In with Ethereum) - Enhanced
- **Endpoints**:
  - `GET /v2/auth/siwe` - Get SIWE message
  - `POST /v2/auth/verify` - Verify signature and get JWT
- **Headers**: `Authorization: Bearer <jwt>`

### Code Comparison

**V1 Authentication:**
```javascript
// Get SIWE message
const siweResponse = await fetch('https://api.rise.works/v1/api/siwe', {
  method: 'GET'
});
const { siwe } = await siweResponse.json();

// Sign and verify
const signature = await signer.signMessage(siwe);
const verifyResponse = await fetch('https://api.rise.works/v1/api/siwe', {
  method: 'POST',
  body: JSON.stringify({ message: siwe, signature })
});
const { token } = await verifyResponse.json();

// Use in requests
const response = await fetch('/v1/teams', {
  headers: { 'X-Auth-Token': token }
});
```

**B2B Authentication:**
```javascript
// Get SIWE message
const siweResponse = await fetch('https://api.rise.works/v2/auth/siwe?wallet=${wallet}&riseid=${riseId}');
const { siwe } = siweResponse.data.data;

// Sign and verify
const signature = await signer.signMessage(siwe);
const nonce = extractNonce(siwe);
const verifyResponse = await fetch('https://api.rise.works/v2/auth/verify', {
  method: 'POST',
  body: JSON.stringify({ message: siwe, sig: signature, nonce })
});
const { jwt } = verifyResponse.data.data;

// Use in requests
const response = await fetch('/v2/teams', {
  headers: { 'Authorization': `Bearer ${jwt}` }
});
```

## Response Format Comparison

### V1 Response Format
```json
{
  "teams": [
    {
      "id": 123,
      "name": "Engineering Team",
      "created_at": "2024-01-01T00:00:00Z"
    }
  ]
}
```

### B2B Response Format
```json
{
  "success": true,
  "data": {
    "teams": [
      {
        "nanoid": "te-abc123def456",
        "name": "Engineering Team",
        "created_at": "2024-01-01T00:00:00Z"
      }
    ]
  }
}
```

## Error Format Comparison

### V1 Error Format
```json
{
  "error": "Team not found",
  "error_code": "TEAM_NOT_FOUND"
}
```

### B2B Error Format
```json
{
  "success": false,
  "data": "Team te-abc123def456 not found"
}
```

## ID System Comparison

### V1 ID System
- **Teams**: Numeric IDs (`123`, `456`, `789`)
- **Users**: Numeric IDs (`talentId`)
- **RiseID**: String format (`rise_id`)

### B2B ID System
- **Teams**: Nanoids (`te-abc123def456`)
- **Users**: Nanoids (`us-ghi789jkl012`)
- **Companies**: Nanoids (`co-def456ghi789`)
- **Payments**: Nanoids (`pa-pay123def456`)

## Migration Impact Assessment

### Low Impact Changes
- **Authentication**: Minor path updates, same SIWE flow
- **Invites**: Path updates, same functionality
- **Basic Operations**: Same core functionality

### Medium Impact Changes
- **Response Format**: Need to handle `{ success, data }` wrapper
- **Error Handling**: Updated error format
- **ID System**: Replace numeric IDs with nanoids

### High Impact Changes
- **Payment Processing**: Two-step process with EIP-712 signing
- **Team Management**: New endpoints and enhanced features
- **Balance Management**: New endpoint structure
